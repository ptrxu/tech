AlarmNet Lynx GSML Cellular Module

Information Can be registered without using 7720P programmer like the 7845GSM cellular backup

Works only with LynxR-I v17 and above panels

Works with Total Connect

Turn off "Power Save" option if having downloading issues

Power 5VDC to 12VDC supplied by LynxR-I

40mA standby

650mA during transmit

Total Connect Enable as

Programming Device Address = 3

Allow Local Programming = Yes

Enable Power Save = Yes

Remote Access = Yes

Keypad Address = 1

Keypad type = Lynx

MultiMode = Depends if Email Notifications are wanted

Disabled = No Notifications

4204 Sourced = Up to 4 Email Notifications

2 4204 Sourced = Up to 8 Email Notifications

MultiMode Address = 7

If using 4204 Sourced enable in *86 and program X-10 devices 9 to 12 in *80

If using 2 4204 Sourced enable in *86 and program X-10 devices 9 to 16 in *80

Check 103 or 1. Control panel not communicating with cellular backup

Fault 103 2. Primary cellular communication path failure (No cellular service)

3. Cellular backup is not registered

Trouble Signals (Sent to by panel to central station)

E339 C803 = ECP Power On Reset (reboot)

E355 C000 = 7847i lost communication with control panel

R355 C000 = 7847i restored communication with control panel

5555 5555 9 = Test

Cellular/Internet (Sent by Cell/Internet unit in Ademco High Speed Express format)

Signals 01 = Fire Alarm/Fire Panic

02 = Burg Alarm/Police Panic

03 = Medical Panic

08 = Cover Tamper/Restore

C1 = AC Failure

C2 = Low Battery

C4 = Watchdog Failure or Reboot

C5 = Status Restore (Communication trouble restore)

T90 = 24 Hr Test/Module Tamper

U00 = Reboot

7720 Messages (When registering)

Registration SUCCESS Registration OK

Registration BAD - Timed Out No response from AlarmNet (Possible Internet trble)

Registration BAD - Pri Sub ID BAD Primary City Code, Station ID or Primary Account bad

Registration BAD - 2nd Sub ID BAD Secondary City Code, Station ID or Primary Account bad

Registration BAD - Pri & 2nd IDs BAD Primary and Secondary City Code, Station ID or Account bad

Registration BAD - Pri ID Need PIN

Registration BAD - 2nd ID Need PIN Replacement PIN issued from AlarmNet

Registration BAD - Pri & 2nd Need PIN to replace an existing AlarmNet unit.

Commands SHIFT A = Display software revision

SHIFT B = 1st screen - Display MAC address (Press * for 2nd screen)

2nd screen - Display SCID SIM card information (Press * for 3rd screen)

3rd screen - Display IMEI ID number assigned to GSM (Press * to go back to 1st screen)

SHIFT C = Display current AlarmNet Time/Date

SHIFT D = Enryption Test

SHIFT E = 1st screen - Display GSM Status (Press * for 2nd screen)

PriRSS = Signal strength (-50 to -100dB, lower is better)

GPRS - Y = GPRS service is available

N = GPRS service is not available

REG - N = Not registered

H = Registered home

S = Searching

D = Registration denied

R = Registered roaming

? = Unknown registration state

2nd screen - Display GSM Status 2 (Press * for 3rd screen)

Cntry = Country Code

Netw = Network Code

LAC = Local Area Code

3rd screen - Display GSM Status 3 (Press * for 4th screen)

Cell = Base Station ID

BaseST = Base Station Antenna Sector

Chan = Control Channel in use

4th screen - 2nd Site Signal Strength (Press * to go back to 1st screen)

SHIFT F = Not Used

SHIFT S = 1st screen - System Status

ECP = Module enabled for ECP mode

Flt = Radio Faults

OK - Normal

G - No GSM, Fault time expired

g - No GSM, Fault time has not expired

SHIFT T = Send Test message to AlarmNet

Sends a test message

SHIFT X = Reset CPU

Restarts GSML

SHIFT Up Arrow = Register GSML with AlarmNet

SHIFT Down Arrow = Register replacement GSML with PIN# from AlarmNet

SHIFT 0 = Force upload of configuration file to AlarmNet servers

This may mean GSML is not initialized yet with GSM network
